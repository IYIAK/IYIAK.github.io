先搭好界面，再拆分成组件

也就是先只是用 html/css 搭建基本的 UI 界面，然后再根据功能划分拆分成不同的组件。这样的好处是界面的逻辑更加清晰。

当然边创建组件边实现每个组件的 UI，最后拼到一起也可以，我就是这么做的。但是这样思路不够清晰，尤其是 css 样式部分，就很混乱。不建议这么干。
