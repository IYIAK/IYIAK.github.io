JavaScript 中的执行上下文

本文简单介绍了执行上下文相关的知识点。

什么是执行上下文？

执行上下文就类似一个对象，由this、作用域链、变量对象 / 活动对象组成。

执行上下文分为 全局执行上下文 和函数执行上下文。全局执行上下文在浏览器环境中就是 window 对象。这里主要介绍函数执行上下文。

执行上下文又分为 创建阶段 和执行阶段 两个阶段。当函数被调用时，会先创建执行上下文，接着将创建好的执行上下文放入调用栈中，此时就进入了执行阶段。

变量对象（Variable Object）由函数的形参、内部的变量申明、内部的函数声明组成。

作用域链

我参考的文章中是这么介绍作用域链的：

1. 函数被调用前，初始化 function fn，fn 有个私有属性[[scope]], 它会被初始化为当前全局的作用域，fn.[[scope]=”globalScope”。

2. 调用函数 fn(1,2)，开始 创建 fn 执行上下文 ，同时 创建作用域链fn.scopeChain = [fn.[[scope]]]，此时作用域链中有全局作用域。

3.fn 活动对象 AO 被初始化后，把活动对象作为变量对象推到作用域链前端，此时 fn.scopeChain = [fn.AO,fn.[[scope]]]，构建完成，此时作用域链中有两个值，一个当前活动对象，一个全局作用域。

但是问题在于，其中的”全局作用域”应该替换成外部作用域（或者叫外层作用域？），因为函数不一定是在全局作用域中定义的。

this 的值

这个其实是 JS 的基础知识了。

全局执行上下文中，如果不是严格模式，this 指向 window 对象，否则为 undefined。

函数执行上下文中，

在《JavaScript 权威指南》中有这么几句话：
1.this 是关键字，不是变量，不是属性名，js 语法不允许给 this 赋值。
2. 关键字 this 没有作用域限制，嵌套的函数不会从调用它的函数中继承 this。
3. 如果嵌套函数作为方法调用，其 this 指向调用它的对象。
4. 如果嵌套函数作为函数调用，其 this 值是 window（非严格模式），或 undefined（严格模式下）。

简单来说，谁调用函数，this 就指向谁。

直接调用函数 foo()，就相当于全局对象调用；Obj.foo() 的形式调用，this 就指向Obj。

此外，foo.bind(),foo.call(),foo.apply()可以修改 this 指向。

ES6 中的箭头函数的 this 的值永远和定义时所在的执行上下文的 this 值相同。
