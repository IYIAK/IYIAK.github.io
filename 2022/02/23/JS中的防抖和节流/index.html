JS 中的防抖和节流

本文介绍 JavaScript 中防抖和节流的简单实现。

防抖

假设一个场景，比如需要记录网页滚动时距顶部的距离

上面这种情况，log 的频率会非常高。

为了避免过于频繁的输出，需要用到防抖。

原理简单来说就是，给事件处理函数设置一个定时器，如果在上一个定时器的倒计时还没结束时再一次触发了事件，就清除上一个定时器，再重新设置一个定时器。这样只要事件不断的连续触发，定时器就会不断的设置 / 清除，就很难等到定时器倒计时结束从而真正执行事件处理函数。只有事件不再触发后，最后一个定时器就不会被清除，等倒计时结束后自然就可以触发事件处理函数了。

这样的效果就是，滚动时不会输出距离，但是停止滚动后，等一秒钟就会输出当前据顶部距离。

节流

目的也防止过于频繁的事件触发，但是和防抖的区别在于，防抖是只要不断触发事件就不会执行事件处理函数，节流是需要以 一定的间隔 执行事件处理函数，即使事件仍在被不断触发。

还是以页面滚动为例

实现思路其实和防抖差不多，都是利用了闭包。只是这次是闭包了一个有效值 valid，当事件触发且valid 为true时，就将 valid 设为 false 并执行定时器，否则就直接返回。这样便保证了不会连续触发多个定时器。定时器倒计时结束便执行事件处理函数并再将 valid 设为true，这样下次触发时便可以再次设置定时器了。
